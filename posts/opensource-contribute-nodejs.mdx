## 기여를 도전한 계기

2024 오픈소스 컨트리뷰션 아카데미에서 Node.js 팀에 선발이 되었습니다. 이전에도 react-icons, eslint-plugin-react 등에 기여한 경험이 있었기 때문에
해당 대외활동을 지원하는데에 크게 망설임은 없었고 새로운 지식과 더불어서 멘토님과 팀원분들과 활동을 하며 시야를 넓히고자 했습니다.

## Node.js 빌드

첫 활동으로 Node.js 프로젝트는 빌드해보는 시간을 가졌습니다. 또한 멘토링으로 Node.js 빌드 프로세스에 대한 교육을 들었는데 내용이 친숙하진 않았고
난이도도 쉽지 않았던 것으로 기억합니다. 기억에 남는 것은 Node.js 빌드를 할때 윈도우 기반 노트북에서는 빌드 속도가 매우 느렸던 것으로 기억합니다. 1시간은 더 걸렸던 것으로 기억하고 그에 반해
M3칩이 탑재된 제 Macbook으로는 빌드시간이 20분은 안걸렸던 것으로 기억합니다.

## 기여 포인트 찾기

본격적인 기여활동이 시작하고 다른 팀원분들과 마찬가지로 Node.js 프로젝트 내에서 기여할만한 포인트를 찾기 시작했습니다. 처음에 잡은 방향으로는
문서 typo 수정 혹은 개선, Javascript 레이어에서의 API 개선 등이 있었습니다. 아직 다른 분들이 어떤 기여를 하시는지 몰랐기에 눈에 불을 켜고 여러 코드들을 살펴보았습니다.
Javascript로 작성된 API 함수들에서 개선점을 여러개 발견하기도 했고, 문서에서 typo를 찾기도 하였습니다. typo를 찾을때 프로그램을 돌리시는 분도 있는데 저는 100% 수작업으로 typo를 찾아내었습니다.

## Node.js에 첫 기여를 하다

typo를 찾아내고 나서 이를 고친 후, Pull Request를 보냈습니다. 얼마 지나지 않아서 2명의 리뷰어 분들이 Approved를 주셨고, 이후에도 몇몇의 Approved를 추가로 받으며 Merge가 되었습니다.
대단한 기여를 하지는 않았지만 Node.js에 기여를 했다는 것에 스스로 뿌듯함을 느꼈습니다.

## 어려움을 느꼈던 부분

이 이외에도 기여를 몇 번 시도했습니다. 주로 Javascript로 작성된 API 함수들의 동작을 개선하는 기여를 했는데 Merge가 되지는 않았습니다.
멘토링을 통해서 여러 기여포인트들도 알게 되었고 시도도 해보았지만 실패도 했습니다. 예를들어 API 성능개선을 위해 코드를 수정했지만 오히려 성능이 더 안좋아지는 경우도 있었습니다.
기여 난이도가 대체적으로 쉽지는 않았고 대학교 수업과 다른 대외활동도 병행하며 시간을 투자하는데도 어려움을 겪었습니다.

## 두번째 기여 및 총 기여 횟수

멘토링이나 팀활동에는 적극적으로 참여하고 적지만 조금이라도 기여할 수 있는 시간을 가지고자 하였습니다. 결과적으로 typo 수정 외에 Javascript로 작성된 함수의 인자에 대한 validation 과정을 추가하는 작업을 하여 Pull Request를 생성하고 Merge가 되었습니다.
저는 총 6개의 Pull Request를 생성했고, 총 2개가 Merge가 되었습니다.

## 느낀점

Node.js 프로젝트가 얼마나 잘 관리되고 있는지 느꼈습니다. 문서화가 정말 잘 되어있고 메인테이너들이 얼마나 체계적으로 프로젝트를 관리하고 있는지 알 수 있었습니다.
활동을 하며 어려운 부분들이 많았고 시간을 더 투자하지 못한 것도 아쉽습니다. 하지만 활동을 하며 시야가 넓어졌으며 Node.js라는 큰 프로젝트에 2번이나 기여를 했다는게 스스로 정말 뿌듯한 부분입니다.
앞으로 오픈소스 기여를 계속 하며 새로운 것을 배우고 성장하고 싶습니다.
